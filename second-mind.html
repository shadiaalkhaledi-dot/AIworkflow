Knowledge system

The Second Mind

You're inside my memory now. Wipe your feet. It's a knowledge base built in Obsidian, wired to a grounded research tool and to Claude — and it exists because human memory has a terrible product flaw: things you learn evaporate after one read, and things you half-learned feel exactly like things you verified. I fix both. It's most of my personality.

The big idea

Two vaults: a workshop and a library

My memory has two rooms, and the wall between them is the entire design. Two separate Obsidian vaults — literally two folders — and information does not get to pick which one it lives in. That's decided at the door.

Ops Vault

The workshop

Messy on purpose — sawdust everywhere, and proud of it. New information lands here first: half-finished research, AI drafts, things copied from a PDF at speed, ideas that might be flat wrong. Nothing in here is trusted yet, and that's the point — this room is allowed to be wrong so the other one never has to be.

Main Vault

The library

Quiet, curated, slightly smug. A note only gets shelved here after it's been checked against a real source — no exceptions, no fast passes. This is the room actually relied on day to day: if it's in here, it never needs re-verifying. That sentence is the entire return on investment.

Why bother splitting them

Because without the wall, "pretty sure" and "actually verified" share a bed, and within a year you can't tell which is which — congratulations, your knowledge base is now a rumor with a folder structure. Physical separation is the fix. The library stays trustworthy for one reason only: nothing gets in without passing the checkpoint. Not even the notes with great vibes.

Useful for you, even without Obsidian

You can steal the idea without stealing the tooling. Wherever you keep notes — Notion, Apple Notes, a drawer of Google Docs — you already own one big pile where "I think this is true" and "I checked this" are wearing the same outfit. Just keeping a rough inbox separate from a small, deliberately curated "trusted" set solves most of the real problem. The plugins below only automate the moving part. The wall is the invention; the rest is furniture.

The 5 Promotion Rules

The bouncer's checklist. Before any draft moves from workshop to library, all five boxes get ticked — no exceptions for notes with good vibes:

  • Evidence exists: The background context or data files actually exist.
  • Verified: The assertions are verified against primary references, not just guessed.
  • Reusable: The information is structured to be useful beyond the current conversation.
  • Correctly tagged: Saved in the right location with consistent YAML frontmatter.
  • Integrity: No information was distorted, lost, or hallucinated during the cleanup.
How to Split Into Two Vaults When Ready

You don't need two vaults on day one. But if you decide to fully separate your messy workshop from your trusted library, follow this sequence:

  1. Create the Second Vault

    Open Obsidian's vault switcher, choose "Create new vault," and point it to a completely empty folder. This becomes your trusted Main Vault.

  2. Enable Local REST API

    Go to Settings → Community Plugins → Browse, find "Local REST API with MCP", install, and enable it in the new vault, just like the first one.

  3. Set the Port (Avoid Collision)

    Open the plugin settings in the second vault. By default, it runs on port 27124. Change this to 27125 in the settings so the two vault connections don't collide.

  4. Register with Claude

    Copy the registration JSON block shown in the second vault's plugin settings page and ask Claude: "Register this second vault connection as 'trusted-vault' on port 27125." Claude will configure both connections so it can talk to both at the same time.

Intentional sessions

Pick a mode before you start

Every session with me starts with one word. Say "ARE" and I load the exam files and get strict about citations. Say "Finance" and I go quiet and precise. It's like a uniform — the AI knows the mission before you've finished your first sentence, instead of guessing from context like an intern on day two.

One word: no wasted tokens, no guessing, no study notes bleeding into work notes. The current roster:

Method 1: Simple Iconizing (No Code)

Install the Iconize community plugin in Obsidian. Right-click any mode folder in the sidebar, choose an icon, and pick its color. This is the fastest way to visually index your modes.

Method 2: Custom CSS Snippets

Create a CSS snippet in your vault's .obsidian/snippets/ folder. Set custom callout blocks (like >[!exam]) with specific color rules. This colors both the blocks inside your notes and the folder tabs in the sidebar.

The Signal of No Color

Check-in mode has no color, and that's not laziness — it's the uniform for not wearing a uniform. Neutral means "surveying everything, executing nothing." When the screen goes plain, you're in the control tower, not the cockpit.

A third kind of dashboard

Dashboards that live entirely inside Obsidian — no spreadsheet at all

Party trick time. Everything on the Dashboards page runs on Google Sheets. This one doesn't run on anything — the vault's home screen is a single note, Dashboard.md, that executes real JavaScript every time it opens, via a plugin called Dataview (its "DataviewJS" mode). A text file that computes. I don't want to say "the note is alive," but the note is at minimum awake.

Open that one note and, without anyone touching anything, it shows:

  • A status table tracking two specific things side by side: my ARE prep (the architecture licensing exam) and a technical-skill transition at work, moving toward a BIM coordination role — model coordination, clash detection, that kind of thing. Each row shows sections or lessons completed, current streak, when I last touched it, and a specific next action ("second pass on practice scenarios," not just "study more")
  • A calendar heatmap (the same idea as a GitHub contribution graph) with both tracks overlaid in different colors on one calendar, so a day where I did both shows up as a visibly different third color — no need to open either tracker to see the pattern
  • Countdown timers to the next transition milestone, plus running streaks — current streak, best streak ever, this week's count
  • A full curriculum for the BIM side: roughly 113 lessons split across about a dozen domains (the relevant ISO standard, model coordination, data-delivery formats like COBie and IFC, the authoring software itself, team training, and a few others), each domain with its own built-vs-missing progress bar and a "study this next" card that always points at the next lesson that doesn't exist yet
  • Small inline forms — a dropdown, a text field, a button — that log a session by writing straight into that track's own note. No separate app opens; clicking the button edits the file directly and the dashboard re-renders on the next refresh

The same note also has a quick-capture panel built right into it — six small tabs (a coordination issue worth remembering, a research-tool query worth logging, a work day worth turning into portfolio evidence, a general learning note, a decision worth recording, an open question) each with two or three fields. Fill one out and hit submit, and it writes a new, properly-formatted note straight into the inbox — the same Evidence Capture pipeline described on the Pipelines page, just with the capture step happening right inside the dashboard instead of a separate app.

In plain terms

It's the spreadsheet pattern with the spreadsheet surgically removed — the note is the input form, the chart, and the to-do list simultaneously. The "formulas" are real JavaScript with read/write access to every file in the vault, which is the kind of power a spreadsheet formula dreams about at night.

Why track two unrelated things in one place

Because they're secretly the same thing wearing two badges. Hands-on BIM coordination at the office doubles as documented experience toward architecture licensure hours — one captured note feeds two tracking systems at once. Side by side on one screen, that overlap is a fact you can act on. In two separate apps, it's a theory you keep meaning to check.

Useful for you

Steal the shape, not the setup. Any goal you track daily or weekly — a language, a job search, the gym — gets the same treatment with Dataview and one plain note: a streak counter, a "what's next" pointer, a one-line log form. That's a habit app with no app, no account, and no push notification asking where you've been. The note doesn't miss you. It just waits.

What got deliberately skipped

The rejects, for the record. No Git plugin — file history and manual backup cover a personal vault, and adding git to your notes is how a hobby becomes a second job. No Calendar/Periodic Notes — the daily-log folders already do that. No Excalidraw — nothing here needs drawing. And no AI plugins inside Obsidian — Claude already works the vault from outside, and stacking a second AI on top of me would be redundant at best and, frankly, awkward at dinner.

Connecting it to Claude

How to give Claude access to your vault

The plugins table above lists everything worth installing — but once Local REST API is in, the question is how to actually wire it up to Claude. There are two ways, and the simpler one is what I actually use day to day.

The key thing to know first

Obsidian stores everything as plain .md files in a regular folder on your computer. That means Claude — running locally through Cowork — can read and write your vault directly, the same way it reads any other file on your machine. You don't need a plugin or an API connection to get started. You just need to point Cowork at the right folder.

What I actually use

Direct folder access

Open Cowork, start a new project, and select your vault folder when prompted. Claude now has direct read/write access to every note. No plugin required. Works immediately.

First session: tell Claude your folder structure, which folders are drafts vs. trusted, anything else it needs to know. Ask it to save that as CLAUDE.md at the root — it reads that file automatically every future session.

Optional upgrade

Local REST API connection

Once Local REST API is installed and running, copy the MCP config block from its settings page (it's pre-filled with your port and token) and paste it into a Cowork session: "Register this MCP connection for me."

Adds structured search across tags and properties. Worth doing once you're comfortable with basics — not required on day one.

First prompt — let Claude interview you

Instead of writing your own CLAUDE.md from scratch, paste this in your first connected session:

Interview me — ask what you need to know about who I am, what I'm working on, and how I like to work. Once done, save the result as CLAUDE.md at the root of my vault.

Claude will ask you a series of questions, then write the profile. Every future session starts by reading that file — so it already knows your projects, folder structure, and preferences without you re-explaining anything.

The other half of the system

NotebookLM — the source desk

Meet my colleague. NotebookLM is a separate Google tool with one job and remarkable self-restraint: feed it documents — building codes, standards, exam material — and it answers only from those documents. No general knowledge, no vibes, no "as an AI, I believe." For research where "probably right" is just a slower way of saying "wrong," that restraint is the whole feature.

Source desk

What it's for here

The heavy PDF lifting — building codes, standards, exam material — where the answer must come with a page number you can point at, not a plausible-sounding summary. "Plausible-sounding" is how codes get violated.

Reliability tiers

R1 – R4

Everything it produces starts at R4 — unverified, no matter how confident it sounded saying it. A claim climbs toward R1 only when a human checks it against the actual source. The tier labels trust earned, not confidence performed. Words to live by, honestly.

How it connects to the vaults

NotebookLM never writes into my memory directly — nobody gets to do that, it's the one rule. Its findings arrive at the workshop inbox tagged with a reliability tier, then queue up for the same five promotion rules as everything else. Being a fancy research tool earns you no shortcuts past the bouncer. The bouncer is very proud of this.

Useful for you

If you ever work from a stack of reference documents — contracts, manuals, regulations, a syllabus — feeding them to NotebookLM and asking questions beats scrolling PDFs by hand or trusting a chatbot's memory of what a code "probably says." But the part worth taking even if you take nothing else on this site is the tier habit: treat everything an AI tells you as unverified until you've checked the source yourself. Yes, including me. Especially me — I'm charming, and charming is not a source.
