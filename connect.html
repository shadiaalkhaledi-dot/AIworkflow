Getting started

Setup Guide

How to give Claude access to your Obsidian vault — and why it's simpler than it sounds. Your vault is just a folder of text files. No API, no plugin, no server required to get started.

The key insight

Obsidian stores everything as plain .md files in a regular folder on your computer. There is no database, no cloud, no special format. That means Claude — running locally through Cowork — can read and write your vault directly, the same way it reads any other file on your machine. You don't need a plugin or an API connection to get this working. You just need to point Cowork at the right folder.

The simple path

Direct folder access — what I actually use

This is the method that works best for day-to-day use. Claude reads and writes your vault files directly — no plugin required, nothing to configure in Obsidian.

  1. Install Claude Cowork (or Claude Code)

    Download and install Claude Cowork from the Anthropic website, or use Claude Code if you're comfortable with a terminal. Either gives Claude the ability to read and write files on your local machine — something the browser-based Claude.ai chat cannot do.

  2. Open your vault folder in Cowork

    When you start a new Cowork project, you'll be asked to select a folder. Choose your Obsidian vault folder — the one Obsidian opens when you launch it. That's all. Claude now has direct access to every note in the vault.

  3. Tell Claude what's in there

    In your first session, give Claude a quick orientation. Tell it the folder structure you use, which folders are drafts vs. trusted, and anything else it should know. Then ask it to save that as a file called CLAUDE.md at the root of your vault — Claude will read this automatically at the start of every future session.

    Shortcut

    Don't want to write the orientation yourself? Paste this into your first session and let Claude interview you instead:

    Interview me — ask what you need to know about who I am, what I'm working on, and how I like to work. Once done, save the result as CLAUDE.md at the root of my vault.
  4. Verify it's working

    Ask Claude: "List the files in my vault" or "What's in my Drafts folder?" If it returns real file names from your vault, you're connected. From here, you can ask Claude to read notes, write new ones, search across your vault, or update existing files — all in plain conversation.

Why this is better than an API for most people

I tested the REST API plugin approach (covered below) and it works — but it adds a plugin to install, a port to keep open, and an authorization token to manage. For reading and writing notes, direct folder access is faster to set up and just as capable. The API is worth adding later if you want Claude to search across the vault more intelligently, but you don't need it on day one.

Optional upgrade

The REST API plugin — when to add it

The Obsidian Local REST API plugin gives Claude a more structured way to interact with your vault — searching by tag, querying properties, accessing the Obsidian graph. Worth adding once you're comfortable with the basics, but not a requirement to get started.

  1. Install the plugin

    In Obsidian: Settings → Community Plugins → Browse → search for "Local REST API with MCP" (by Adam Coddington) → install and enable it. The plugin runs a small local server on your machine. It does not send anything outside your computer.

  2. Copy your connection details

    Open the plugin settings in Obsidian. You'll find an auto-generated configuration block pre-filled with your port and authorization token:

    { "mcpServers": { "obsidian": { "type": "http", "url": "https://127.0.0.1:27124/mcp/", "headers": { "Authorization": "Bearer your-api-key-here" } } } }

    Copy this block directly from the plugin page. Don't hand-type your token.

  3. Register the connection

    Paste the copied JSON into a Cowork or Claude Code session and say: "Register this MCP connection for me." Claude will configure it and confirm. Restart your session, then test with: "Search my vault for notes tagged #decision."

Two vaults?

If you're running the two-vault setup described on the Second Mind page, you'll install the plugin in both vaults and set the second one to port 27125 so they don't collide. Register both connections with different names (e.g. obsidian-ops and obsidian-main). From that point, Claude can read and write to either vault in the same session.
